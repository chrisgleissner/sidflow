# Plan archives

`PLANS.md` is the active planning surface for SIDFlow. Historical tasks should not accumulate there indefinitely.

Archive rules:

- Move completed, superseded, blocked-without-near-term-action, or no-longer-needed tasks into `doc/plans/archive-*.md`.
- Preserve enough context to recover intent: request summary, status, key progress notes, and validation evidence.
- If archived work becomes active again, do not copy it back wholesale. Reopen it as part of the current active roadmap and link to the archived entry.
- Prefer chronological archive files, for example `archive-2026-03.md` or `archive-2025-12-to-2026-03.md`.
